# Database Documentation

## Overview

This document describes the database schema for the Sri Raghavendra Swamy Matha temple management system, built with Prisma ORM and PostgreSQL.

## Architecture

### Technology Stack
- **ORM**: Prisma v7.8.0
- **Database**: PostgreSQL
- **Schema Location**: `prisma/schema.prisma`
- **Migrations**: `prisma/migrations/`

### Domain Model Version
This schema implements **Domain Model v5** as documented in `docs/DOMAIN_MODEL.md`.

## Migrations

### Initial Schema Migration

**Migration ID**: `20250101000000_initial_schema`

This migration creates all tables, enums, indexes, and constraints for the initial schema.

### Running Migrations

```bash
# Development (creates migration if needed)
npm run db:migrate

# Production (applies existing migrations)
npm run db:migrate:deploy

# Reset database (DANGER: drops all data)
npm run db:reset
```

### Migration Commands

```bash
# Check migration status
npx prisma migrate status

# Validate schema
npx prisma validate

# Generate Prisma Client
npx prisma generate
```

## Database Conventions

### Primary Keys
- All tables use UUID primary keys (`id` field)
- UUIDs are generated by the application using `crypto.randomUUID()`

### Audit Fields
- `createdAt`: Timestamp when record was created
- `updatedAt`: Timestamp when record was last modified
- `deletedAt`: Soft delete timestamp (null means active)

### Naming Conventions
- Table names: plural, snake_case (e.g., `temple_days`)
- Column names: snake_case (e.g., `created_at`)
- Enum names: PascalCase (e.g., `BookingStatus`)

### Indexes
- Primary keys: unique index on `id`
- Foreign keys: indexed for join performance
- Unique constraints: on fields requiring uniqueness
- Composite indexes: on frequently queried field combinations

## Row Level Security (RLS)

RLS policies are defined in `prisma/rls-policies.sql`.

## Storage

Media files are stored in Supabase Storage buckets defined in `prisma/storage-buckets.sql`.

### Default Buckets
- `media` - General media files
- `gallery` - Gallery images
- `documents` - Downloadable documents

## API Routes

All API routes are located in `app/api/` and use the Prisma Client for database access.

### Database Client Usage

```typescript
import { prisma } from '@/lib/db'

// Example query
const announcements = await prisma.announcement.findMany({
  where: { isActive: true },
  orderBy: { createdAt: 'desc' }
})
```

## Troubleshooting

### Migration Failures

If migrations fail, check:
1. Database connection is working
2. User has sufficient permissions
3. No conflicting migrations exist

### Seed Failures

If seeding fails:
1. Ensure migrations have been applied
2. Check for duplicate data
3. Verify database connection
